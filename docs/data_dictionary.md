## 1. Tables <a name="tables"></a>

### `sales_reps` <a name="sales_reps"></a>

Master dimension table for sales representatives.

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `rep_id` | INTEGER | No | Surrogate primary key. Auto-assigned sequential integer starting at 1. |
| `rep_name` | TEXT | No | Full name of the sales representative (synthetic, generated by Faker). |
| `region` | TEXT | No | Geographic sales region. One of: `North America`, `EMEA`, `APAC`, `LATAM`. |
| `role` | TEXT | No | Job title / role. One of: `Enterprise AE`, `Mid-Market AE`, `SMB AE`, `SDR`, `Sales Manager`. |
| `quota` | REAL | No | Annual revenue quota in USD. Value is rounded to the nearest $5,000. Must be > 0. |
| `hire_date` | TEXT | No | Date the rep joined the company, formatted `YYYY-MM-DD`. Range: 2019-01-01 to 2023-12-31. |

**Row count:** 100 (default)
**Primary key:** `rep_id`

---

### `sales_transactions` <a name="sales_transactions"></a>

Fact table containing individual closed-won sales deals.

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `transaction_id` | INTEGER | No | Surrogate primary key. Auto-assigned sequential integer starting at 1. |
| `rep_id` | INTEGER | No | Foreign key → `sales_reps.rep_id`. Identifies which rep closed the deal. |
| `deal_date` | TEXT | No | Date the deal was closed, formatted `YYYY-MM-DD`. Range: 2024-01-01 to 2024-12-31. |
| `deal_amount` | REAL | No | USD value of the closed deal. Log-normally distributed; minimum $500. |
| `product_category` | TEXT | No | Product line sold. One of: `Enterprise Software`, `Cloud Services`, `Professional Services`, `Hardware`, `Support & Maintenance`. |
| `customer_segment` | TEXT | No | Buyer segment. One of: `Enterprise`, `Mid-Market`, `SMB`, `Government`. |

**Row count:** 12,000 (default)
**Primary key:** `transaction_id`
**Foreign key:** `rep_id → sales_reps.rep_id`

**Distribution notes:**
- Deal amounts follow a log-normal distribution (σ = 0.8).
- The log-normal mean is scaled per role: Enterprise AEs average 3× the base
  mean; SDRs average 0.4× the base mean.
- Deals are uniformly distributed across calendar year 2024.

### `incentive_plan` <a name="incentive_plan"></a>

Reference / slowly-changing dimension describing the commission structure.
One row per (role, tier) combination.

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `plan_id` | INTEGER | No | Surrogate primary key. Sequential integer. |
| `role` | TEXT | No | Sales role this tier applies to. |
| `threshold_min` | REAL | No | Lower bound of the attainment band (quota fraction, inclusive). E.g. 0.5 = 50% of quota. |
| `threshold_max` | REAL | No | Upper bound of the attainment band (quota fraction, exclusive for intermediate tiers). |
| `commission_rate` | REAL | No | Commission rate applied to revenue earned within this band. Decimal in [0, 1]. |
| `accelerator_rate` | REAL | No | Accelerator rate applied on incremental revenue above the accelerator threshold. Decimal in [0, 1]. |
| `effective_from` | TEXT | No | ISO date from which this plan version is effective. Format: `YYYY-MM-DD`. |

**Row count:** 20 (5 roles × 4 tiers)
**Primary key:** `plan_id`

---

### `calendar` <a name="calendar"></a>

Date dimension table providing pre-computed time-intelligence attributes
for every calendar day in the analysis period.

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `date` | TEXT | No | Calendar date in ISO-8601 format `YYYY-MM-DD`. Primary key. |
| `year` | INTEGER | No | Four-digit calendar year (e.g. 2024). |
| `quarter` | INTEGER | No | Calendar quarter: 1–4. |
| `month` | INTEGER | No | Calendar month: 1–12. |
| `month_name` | TEXT | No | Full English month name (e.g. "January"). |
| `week` | INTEGER | No | ISO 8601 week number: 1–53. |
| `day_of_week` | TEXT | No | Full English day name (e.g. "Monday"). |
| `is_weekend` | INTEGER | No | 1 if Saturday or Sunday, else 0. |
| `is_month_end` | INTEGER | No | 1 if the last day of the month, else 0. |
| `is_quarter_end` | INTEGER | No | 1 if the last day of the quarter (Mar 31, Jun 30, Sep 30, Dec 31), else 0. |

**Row count:** 366 (2024 is a leap year)
**Primary key:** `date`

### `payout_results` <a name="payout_results"></a>

Output table produced by the incentive calculation engine.  Refreshed on
every execution of `src/main.py`.

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `rep_id` | INTEGER | No | Primary key, foreign key → `sales_reps.rep_id`. |
| `rep_name` | TEXT | Yes | Denormalised name from `sales_reps` for reporting convenience. |
| `region` | TEXT | Yes | Denormalised region from `sales_reps`. |
| `role` | TEXT | Yes | Denormalised role from `sales_reps`. |
| `total_sales` | REAL | Yes | Total USD revenue closed by this rep in the analysis period. |
| `quota` | REAL | Yes | Annual quota for this rep (sourced from `sales_reps`). |
| `attainment_pct` | REAL | Yes | `total_sales / quota`. A value of 1.0 = 100% attainment. |
| `base_commission` | REAL | Yes | Commission earned from the tiered structure, before accelerator. |
| `accelerator_bonus` | REAL | Yes | Additional commission earned on incremental revenue above the accelerator threshold. |
| `total_payout` | REAL | Yes | `base_commission + accelerator_bonus`. |
| `payout_to_revenue_ratio` | REAL | Yes | `total_payout / total_sales`. Measures the cost efficiency of the plan. |

**Row count:** 100 (one row per rep)
**Primary key:** `rep_id`

## 2. Analytical Views <a name="views"></a>

### `rep_performance_summary` <a name="rep_performance_summary"></a>

Aggregated performance metrics for each rep with intra-cohort rankings.

| Column | Source | Description |
|--------|--------|-------------|
| `rep_id` | `sales_reps.rep_id` | Rep identifier. |
| `rep_name` | `sales_reps.rep_name` | Rep full name. |
| `region` | `sales_reps.region` | Geographic region. |
| `role` | `sales_reps.role` | Job role. |
| `quota` | `sales_reps.quota` | Annual quota. |
| `total_sales` | `SUM(deal_amount)` | Total revenue; 0 for reps without transactions. |
| `attainment_pct` | Derived | `total_sales / quota × 100` (expressed as a percentage integer here). |
| `deal_count` | `COUNT(transaction_id)` | Number of closed deals. |
| `avg_deal_size` | `AVG(deal_amount)` | Mean deal size. |
| `region_rank` | `RANK() OVER (PARTITION BY region)` | Rank within region by total sales (1 = best). |
| `role_rank` | `RANK() OVER (PARTITION BY role)` | Rank within role by total sales. |
| `percentile_rank` | `PERCENT_RANK() OVER ()` | Global percentile rank (0 = best, 100 = worst); expressed as 0–100. |

---

### `monthly_payout_trend` <a name="monthly_payout_trend"></a>

Monthly revenue aggregation with cumulative totals and month-over-month growth.

| Column | Description |
|--------|-------------|
| `month` | Year-month string `YYYY-MM`. |
| `region` | Geographic region. |
| `role` | Job role. |
| `monthly_revenue` | Sum of `deal_amount` for this month/region/role combination. |
| `deal_count` | Number of deals closed this month. |
| `cumulative_revenue` | Running total of `monthly_revenue` within the region, ordered by month. |
| `prev_month_revenue` | Previous month's revenue for the same region/role (LAG of 1). |
| `mom_growth_pct` | Month-over-month revenue growth: `(current − prior) / prior × 100`. NULL for the first month. |

---

### `region_wise_efficiency` <a name="region_wise_efficiency"></a>

Region-level aggregation comparing total quota to total revenue.

| Column | Description |
|--------|-------------|
| `region` | Geographic region. |
| `rep_count` | Number of distinct reps in this region. |
| `total_quota` | Sum of all rep quotas in this region. |
| `total_revenue` | Sum of all deal amounts attributed to reps in this region. |
| `region_attainment_pct` | `total_revenue / total_quota × 100`. |
| `avg_rep_attainment_pct` | Average of individual rep attainment percentages (equal-weighted). |
| `reps_above_quota` | Count of reps whose total sales ≥ their quota. |
| `pct_reps_above_quota` | `reps_above_quota / rep_count × 100`. |

---

### `over_quota_analysis` <a name="over_quota_analysis"></a>

Rep-level attainment with performance-band labelling for at-risk and
high-payout identification.

| Column | Description |
|--------|-------------|
| `rep_id` | Rep identifier. |
| `rep_name` | Rep full name. |
| `region` | Geographic region. |
| `role` | Job role. |
| `quota` | Annual quota. |
| `total_sales` | Total revenue (0 if no transactions). |
| `attainment_pct` | `total_sales / quota × 100`. |
| `performance_band` | One of: `High Risk – 150%+`, `Above Quota`, `On Track`, `At Risk`. |
| `quartile` | Quartile rank (1 = top 25%, 4 = bottom 25%) by attainment, computed using `NTILE(4)`. |
| `total_payout` | From `payout_results.total_payout`. |
| `payout_to_revenue_ratio` | From `payout_results.payout_to_revenue_ratio`. |

## 3. Business Glossary <a name="glossary"></a>

| Term | Definition |
|------|------------|
| **Quota** | The annual revenue target assigned to a sales rep. Expressed in USD. |
| **Attainment** | `total_sales / quota`. A decimal fraction where 1.0 = 100% (at quota). |
| **Tier** | A contiguous range of attainment within which a fixed commission rate applies to the revenue earned in that band. |
| **Tiered Commission** | A commission structure where the rate changes at defined attainment thresholds. The rate applies only to incremental revenue within each band, not to total revenue. |
| **Base Commission** | The sum of commission earned across all tiers, before the accelerator bonus is applied. |
| **Accelerator** | A higher commission rate applied to incremental revenue above a defined attainment threshold (default: 100% of quota). The accelerator is additive to the base commission. |
| **Accelerator Bonus** | The dollar value earned from the accelerator: `(total_sales − quota × threshold) × accelerator_rate`. |
| **Total Payout** | `base_commission + accelerator_bonus`. |
| **Payout-to-Revenue Ratio** | `total_payout / total_sales`. The cost of the incentive plan as a percentage of revenue; used for budgeting and plan efficiency analysis. |
| **AE (Account Executive)** | A quota-carrying field sales role responsible for closing new business. |
| **SDR (Sales Development Rep)** | A sales role focused on pipeline generation; carries a lower, activity-based quota. |
| **Sales Manager** | A player/coach role with a team quota that aggregates individual quotas. |
| **Performance Band** | A categorical label grouping reps by attainment: At Risk (<50%), On Track (50–99%), Above Quota (100–149%), High Risk – 150%+. |
| **What-if Simulation** | A parameterised re-run of the incentive engine with modified inputs (e.g. higher quotas, different rates) to model alternative plan outcomes without touching the source data. |
