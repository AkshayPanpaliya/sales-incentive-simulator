# Power BI Dashboard – Sales Incentive Compensation Simulator

This document describes the recommended Power BI report design for the Sales
Incentive Compensation Simulator.  The report connects to the SQLite database
generated by `src/main.py` via the **ODBC** or **SQLite ODBC Driver** connector,
or alternatively to the exported CSV files in `data/`.

## 1. Connecting to the Data Source <a name="connecting"></a>

### Option A – SQLite via ODBC
1. Install **SQLite ODBC Driver** from http://www.ch-werner.de/sqliteodbc/
2. In Power BI Desktop: **Get Data → ODBC**
3. DSN string: `Driver={SQLite3 ODBC Driver};Database=<abs_path>/data/sales_incentive.db`
4. Import the following tables / views:
   - `sales_reps`
   - `sales_transactions`
   - `payout_results`
   - `rep_performance_summary` (view)
   - `monthly_payout_trend` (view)
   - `region_wise_efficiency` (view)
   - `over_quota_analysis` (view)

### Option B – CSV Import
1. **Get Data → Text/CSV**
2. Import each file from `data/`:
   - `sales_reps.csv`
   - `transactions.csv`
   - `payout_results.csv`
3. Create relationships manually in the Model view (see §2).

## 2. Data Model <a name="data-model"></a>

```
calendar (date PK)
    │
    └── sales_transactions (deal_date FK)
            │
            └── sales_reps (rep_id PK)
                    │
                    └── payout_results (rep_id FK)
                    │
                    └── incentive_plan (role FK)
```

**Relationships (all 1-to-many, single direction):**

| From (1)               | To (many)              | Join column |
|------------------------|------------------------|-------------|
| `sales_reps.rep_id`    | `sales_transactions.rep_id` | rep_id |
| `sales_reps.rep_id`    | `payout_results.rep_id`     | rep_id |
| `calendar.date`        | `sales_transactions.deal_date` | date |

## 3. Page 1 – Executive Overview <a name="page-1"></a>

**Purpose:** Single-screen C-suite summary of the company's incentive spend
and revenue performance for the current period.

### KPI Cards (top row)
| Card | Measure | Format |
|------|---------|--------|
| Total Revenue | `[Total Revenue]` | $M |
| Total Incentive Payout | `[Total Payout]` | $M |
| Payout-to-Revenue Ratio | `[Payout Ratio %]` | % |
| Avg Rep Attainment | `[Avg Attainment %]` | % |
| % Reps Above Quota | `[% Above Quota]` | % |

### Visuals
- **Clustered Bar Chart** – Revenue vs. Quota by Region (from `region_wise_efficiency`)
- **Donut Chart** – Revenue mix by `product_category`
- **Line Chart** – Monthly revenue trend with prior-year comparison trendline
  (source: `monthly_payout_trend`, x-axis = `month`)
- **Table** – Top 10 reps by total payout, columns:
  `rep_name | region | role | total_sales | total_payout | attainment_pct`

### Slicers
- Year (from `calendar.year`)
- Region (from `sales_reps.region`)
- Role (from `sales_reps.role`)

## 4. Page 2 – Rep Performance <a name="page-2"></a>

**Purpose:** Detailed drill-down into individual rep performance, ranking,
and deal activity.

### Visuals
- **Scatter Plot** – Attainment % (x-axis) vs. Total Payout (y-axis), bubble
  size = deal count, color = region (source: `rep_performance_summary`)
- **Matrix / Pivot Table** – Reps × Month with deal amount values and
  conditional formatting (green = above quota, amber = 50–99%, red = below 50%)
- **Bar Chart** – Deal count by `product_category`, segmented by `customer_segment`
- **Card** – Selected rep's attainment %, payout, and region rank (drill-through
  from scatter plot)
- **Table** – Full rep list sortable by any KPI column

### Slicers
- Region (multi-select)
- Role (multi-select)
- Attainment Band (calculated column: "At Risk" / "On Track" / "Above Quota" / "Overachiever")
- Hire Date range

## 5. Page 3 – What-if Simulator <a name="page-3"></a>

**Purpose:** Interactive sandbox for finance/RevOps to model alternative
compensation plan scenarios without touching source data.

### Parameters (Power BI What-if Parameters)
| Parameter | Type | Range | Default |
|-----------|------|-------|---------|
| Quota Adjustment % | Decimal | -50% to +50% | 0% |
| Accelerator Rate | Decimal | 5% to 30% | 15% |
| Accelerator Threshold | Decimal | 75% to 125% | 100% |
| Tier 1 Rate (0–50%) | Decimal | 0% to 10% | 2% |
| Tier 2 Rate (50–100%) | Decimal | 0% to 15% | 5% |
| Tier 3 Rate (100–150%) | Decimal | 0% to 20% | 8% |
| Tier 4 Rate (150%+) | Decimal | 0% to 25% | 12% |

### Visuals
- **KPI Cards** – Scenario payout vs. base payout with delta indicators
- **Line Chart** – Payout curve: attainment % (x) vs. total payout (y) for
  both base and scenario overlaid
- **Waterfall Chart** – Breakdown of payout delta by commission tier
- **Table** – Rep-level payout comparison (base vs. scenario, delta, % change)

### Implementation Note
Power BI's native What-if Parameters create a disconnected table with a
slicer. DAX measures consume the parameter value to compute the scenario
results dynamically using `SELECTEDVALUE()`.

## 6. Page 4 – Plan Optimisation <a name="page-4"></a>

**Purpose:** Cost-of-sales analysis and recommendations for plan design.

### Visuals
- **Histogram** – Distribution of rep attainment % (bins: 10% wide)
- **Box Plot** – Payout distribution by role (requires custom visual or R/Python visual)
- **Heatmap Matrix** – Region × Role, coloured by average attainment %
- **Scatter Plot** – Deal size (x) vs. number of deals (y), colour = role –
  reveals outlier reps
- **Line Chart** – Cumulative payout cost as attainment rises (illustrates
  cliff-edge commission risk)
- **Table** – `over_quota_analysis` view: reps in "High Risk – 150%+" band

## 7. DAX Measures Reference <a name="dax"></a>

```dax
-- ── Core revenue & payout ──────────────────────────────────────────────────
Total Revenue =
    SUM(sales_transactions[deal_amount])

Total Payout =
    SUM(payout_results[total_payout])

Total Base Commission =
    SUM(payout_results[base_commission])

Total Accelerator Bonus =
    SUM(payout_results[accelerator_bonus])

Payout Ratio % =
    DIVIDE([Total Payout], [Total Revenue], 0) * 100

-- ── Attainment ──────────────────────────────────────────────────────────────
Avg Attainment % =
    AVERAGE(payout_results[attainment_pct]) * 100

% Above Quota =
    DIVIDE(
        COUNTROWS(FILTER(payout_results, payout_results[attainment_pct] >= 1.0)),
        COUNTROWS(payout_results),
        0
    ) * 100

-- ── Deal metrics ────────────────────────────────────────────────────────────
Avg Deal Size =
    AVERAGE(sales_transactions[deal_amount])

Deal Count =
    COUNTROWS(sales_transactions)

-- ── What-if scenario (example using a parameter named "QuotaAdj") ──────────
Scenario Attainment % =
    DIVIDE(
        [Total Revenue],
        SUM(sales_reps[quota]) * (1 + SELECTEDVALUE(QuotaAdj[QuotaAdj Value], 0)),
        0
    ) * 100

-- ── Period-over-period ───────────────────────────────────────────────────────
MoM Revenue Change % =
    VAR CurrentMonth = [Total Revenue]
    VAR PriorMonth   = CALCULATE([Total Revenue],
                           DATEADD(calendar[date], -1, MONTH))
    RETURN DIVIDE(CurrentMonth - PriorMonth, PriorMonth, BLANK()) * 100

QoQ Revenue Change % =
    VAR CurrentQtr = [Total Revenue]
    VAR PriorQtr   = CALCULATE([Total Revenue],
                           DATEADD(calendar[date], -1, QUARTER))
    RETURN DIVIDE(CurrentQtr - PriorQtr, PriorQtr, BLANK()) * 100
```

## 8. Slicer Configuration <a name="slicers"></a>

| Slicer | Field | Style | Default |
|--------|-------|-------|---------|
| Region | `sales_reps[region]` | Dropdown | All |
| Role | `sales_reps[role]` | Dropdown | All |
| Quarter | `calendar[quarter]` | Button (1–4) | All |
| Month | `calendar[month_name]` | Horizontal tile | All |
| Product Category | `sales_transactions[product_category]` | Multi-select list | All |
| Customer Segment | `sales_transactions[customer_segment]` | Multi-select list | All |
| Attainment Band | Calculated column on `payout_results` | Dropdown | All |

Sync all slicers across Page 1 and Page 2 via **View → Sync slicers**.

## 9. Color Theme <a name="color-theme"></a>

Recommended JSON theme file (paste into Power BI via **View → Themes → Browse**):

```json
{
  "name": "Sales Incentive Simulator",
  "dataColors": [
    "#0F4C81",
    "#2196F3",
    "#4CAF50",
    "#FF9800",
    "#F44336",
    "#9C27B0",
    "#00BCD4",
    "#795548"
  ],
  "background": "#FFFFFF",
  "foreground": "#212121",
  "tableAccent": "#0F4C81",
  "sentiment": {
    "good": "#4CAF50",
    "neutral": "#FF9800",
    "bad": "#F44336"
  }
}
```

**Color semantics:**
- 🟢 `#4CAF50` – Above quota / positive variance
- 🟡 `#FF9800` – On track (50–99%)
- 🔴 `#F44336` – At risk (below 50%)
- 🔵 `#0F4C81` – Primary brand / base scenario
- ⚪ `#2196F3` – Secondary / scenario comparison
